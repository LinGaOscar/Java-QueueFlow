# QueueFlow 架構總覽

## 系統定位

QueueFlow 是一套高併發線上候補排隊系統，設計目標是在短時間大量使用者同時加入的場景下（演唱會售票、限量商品搶購、餐廳候位），仍能保持正確的 FIFO 順序、不重複入列、即時狀態推播。

---

## 技術選型理由

| 技術 | 選用原因 |
|------|---------|
| **Java 21** | Virtual Thread（雖本專案未啟用）、Record、text block 等語法讓程式碼更簡潔 |
| **Spring Boot** | 自動配置、DI 容器、JPA / Redis / WebSocket 整合成本低 |
| **PostgreSQL** | 持久化排隊記錄與稽核歷程，支援複雜查詢與 ACID 交易 |
| **Redis** | 記憶體內操作速度快；Sorted Set 天然維護排序；Lua script 保證原子性 |
| **WebSocket (STOMP)** | 伺服器主動推播，使用者不須輪詢即可即時得知狀態變化 |

## 分層架構

```
┌─────────────────────────────────────┐
│           前端 (HTML / JS)           │  index.html（使用者）
│       SockJS + StompJS              │  admin.html（管理員）
└──────────────┬──────────────────────┘
               │ REST API / WebSocket
┌──────────────▼──────────────────────┐
│        api 層（Controller）          │  QueueController
│  路由、參數解析、回應包裝              │  AdminController
└──────────────┬──────────────────────┘
               │ 呼叫 Service
┌──────────────▼──────────────────────┐
│      application 層（Service）        │  QueueService
│  業務邏輯、交易控制、協調 Redis + DB  │  EventService
└──────┬───────────────────┬──────────┘
       │ Redis 操作          │ JPA 操作
┌──────▼──────┐    ┌───────▼────────┐
│ Redis Store  │    │ JPA Repository │
│ Sorted Set   │    │ PostgreSQL     │
│ Lua Script   │    │ ACID 交易      │
└─────────────┘    └────────────────┘
```

**批次層**（`batch`）：獨立排程，與主流程解耦，定期清除過期記錄。

**即時層**（`realtime`）：`QueueNotificationService` 統一發送 WebSocket 訊息，任何 Service 均可呼叫。

## 雙資料庫分工

```
Redis ──── 排隊主結構（速度優先）
           ├─ queue:{eventId}           → Sorted Set，維護 FIFO 順序
           ├─ queue:seq:{eventId}       → 原子遞增序號
           ├─ queue:user:{eventId}:{userId} → 防重複
           ├─ ratelimit:join:{userId}   → 限流
           └─ admit:lock:{eventId}      → 放行互斥鎖

PostgreSQL ─ 永久記錄（正確性優先）
           ├─ queue_event               → 活動資料
           ├─ queue_entry               → 每筆入列記錄（狀態流轉）
           └─ queue_audit_log           → 操作歷程稽核
```

Redis 負責「現在正在等的人是誰、排第幾」，PostgreSQL 負責「歷史上誰入列過、結果為何」。

## 部署依賴

```
[本機開發]
docker-compose up -d   → 啟動 PostgreSQL:5432 + Redis:6379
mvn spring-boot:run    → 啟動 Spring Boot :8080

[前端入口]
http://localhost:8080/index.html   使用者候補頁
http://localhost:8080/admin.html   管理後台
```
